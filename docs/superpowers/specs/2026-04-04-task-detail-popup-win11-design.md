# 2026-04-04 任务详情弹窗 Win11 风格改造设计

## 背景
当前任务详情弹窗可用，但视觉层次弱，内部信息块缺少统一分组和现代化样式，整体观感偏旧。目标是在不显著放大面板尺寸的前提下，将弹窗调整为接近 Win11 的云母感浮层风格。

## 目标
- 保持当前详情弹窗的大致宽度和信息密度，不做明显放大。
- 将弹窗调整为浅色、低对比、轻分层的 Win11 风格。
- 对内部信息做轻微重排，让信息块更规整，但不改变现有核心功能。
- 保持现有交互入口和操作按钮数量不变。

## 非目标
- 不修改详情弹窗的触发方式、定位方式和显示逻辑。
- 不重构任务数据结构。
- 不扩展新的任务字段。
- 不改动其他对话框或主界面整体设计语言。

## 设计方案

### 1. 外层面板
详情弹窗继续使用当前固定宽度，不扩大整体尺寸。外层面板改为偏白的冷灰底色，使用更统一的圆角和更淡的边框，营造 Win11 浮层的轻云母感。

设计要求：
- 外层背景为浅灰白色，不使用深灰底。
- 外层边框为低对比度浅灰描边。
- 圆角统一，风格比当前更柔和。
- 不增加厚重阴影；如需要层次，只允许非常轻的明度差。

### 2. 标题区
标题区单独成层，包含任务标题和颜色按钮。该区域使用轻微底色分层，与下面的信息区分开，但仍保持紧凑。

设计要求：
- 标题文字保持可读性，字重略高。
- 颜色按钮保留，但样式统一到新面板体系。
- 标题区内边距略收敛，避免面板变高。

### 3. 操作按钮区
保留“目录 / 编辑 / 历史记录 / 删除”四个按钮，不改动功能和顺序。按钮样式改为更接近 Win11 的浅底按钮，圆角更统一，hover 反馈更克制。

设计要求：
- 按钮保持一行排列。
- 使用浅色背景和浅灰边框。
- 删除按钮仍可与其他按钮保持同一视觉体系，不单独做强警示色。
- 按钮内边距控制在紧凑范围内，避免按钮行撑高弹窗。

### 4. 信息卡片分层
弹窗内部信息改为紧凑的小卡片分层，但总高度尽量保持接近当前实现。

信息组织：
- 到期日期：单独一块卡片。
- 状态区：状态、紧急度、重要度放入同一块卡片，内部横向排列。
- 备注：单独一块卡片，保留滚动能力。
- 创建时间：单独一块卡片。

设计要求：
- 每个卡片使用浅底色、细边框和统一圆角。
- 卡片之间间距较小，以控制整体高度。
- 卡片内部 padding 统一，避免视觉松散。
- 若某项数据不存在，则对应卡片不渲染，不保留空白占位。

### 5. 备注区
备注区继续支持多行和滚动，但滚动容器本身也要纳入卡片体系，而不是裸露的滚动区。

设计要求：
- 备注内容区域有统一圆角和边框。
- 保持当前最大高度约束，避免长备注把弹窗拉得过高。
- 富文本换行行为保持现状，不改变备注内容格式。

### 6. 代码结构约束
本次改造应尽量局部化，避免引入不必要的 UI 重构。

建议实现边界：
- 主要修改 `ui/styles.py` 中详情弹窗相关样式定义。
- 在 `core/task_label.py` 中做少量结构调整，为标题区、信息卡片区提供稳定的对象名或容器层级。
- 如需新增样式项，应保持命名清晰并限定作用域到详情弹窗，避免再次引发全局样式污染。

## 测试与验证
- 增加或更新针对详情弹窗结构/样式作用域的测试，至少覆盖新增对象名或样式入口存在性。
- 手动验证详情弹窗在以下场景下显示正常：
  - 有到期日期、有备注。
  - 无到期日期、无备注。
  - 备注内容较长，滚动区域仍保持圆角边框。
- 确认弹窗宽度未明显大于当前实现。
- 确认改动不会影响添加任务、设置、历史等其他对话框样式。

## 风险与控制
- 风险：详情弹窗的样式选择器过宽，可能再次污染其他控件。
  - 控制：所有新样式都限定在 `QFrame#task_detail_popup` 及其命名子控件下。
- 风险：信息卡片化后总高度上升。
  - 控制：减少冗余 padding 和区块间距，保持原有固定宽度。
- 风险：备注滚动区样式与 Fluent 组件默认样式冲突。
  - 控制：优先给外层容器命名并定向样式，不直接扩大 Fluent 通用控件样式作用范围。

## 实施摘要
本次实现应以“云母感浅色浮层 + 紧凑信息卡片”为核心，保留现有功能和尺寸级别，通过局部结构命名与定向样式恢复并强化详情弹窗的现代感与层次感。
