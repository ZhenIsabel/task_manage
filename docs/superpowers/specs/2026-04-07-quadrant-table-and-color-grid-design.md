# 四象限表格高度约束与颜色设置网格设计

## 背景

当前四象限任务面板里的历史记录与已完成任务表格，在记录较多时会随着内容持续增高，弹窗高度也会被带着继续扩张。用户希望表格在内容较少时仍然能自然收缩，但当内容较多时，表格本身只增长到一个上限，之后改为表格内部滚动，并且这个上限不应超过四象限面板可用高度。

设置面板的颜色设置页目前按纵向表单线性排列四个象限的颜色按钮和滑杆。用户希望改为一个 `2 x 2` 的网格，每个格子对应一个象限模块，并保留该象限完整的一组颜色调节控件。

本次设计的目标，是在尽量复用现有 `AdaptiveTextTableWidget`、设置预览和配置读写逻辑的前提下，补上表格高度约束能力，并重排颜色设置页布局。

## 目标

1. 历史记录、已完成任务等表格在内容较少时随内容自然增高。
2. 当表格内容较多时，表格高度停止增长，滚动发生在表格内部而不是继续撑高弹窗。
3. 表格最大高度不超过父级四象限面板的可用高度；若无父级，则使用安全默认高度。
4. 设置面板中的颜色设置页改成 `2 x 2` 网格，每个格子对应一个象限，并保留该象限完整颜色调节组。
5. 保持现有颜色预览、设置保存、配置结构和已存在测试约束不变。

## 非目标

1. 不改动历史记录和已完成任务的数据来源、排序和业务逻辑。
2. 不调整四象限主面板的绘制逻辑和象限配置协议。
3. 不引入新的颜色配置字段，也不改变 `quadrants` 与 `color_ranges` 的存储结构。
4. 不在本设计阶段处理其它弹窗或其它表格的统一视觉重构。

## 设计

### 1. 在通用表格组件内增加“受限高度”能力

扩展 `AdaptiveTextTableWidget`，让它支持根据当前内容高度和外部传入的最大高度，自动更新自身显示高度。

该能力的行为规则如下：

- 表格总高度以“表头高度 + 所有行高度 + 边框/边距补偿”为基础计算。
- 若未设置高度上限，则保持现有行为，由布局系统正常管理。
- 若设置了高度上限，则表格实际高度取“内容所需高度”和“高度上限”中的较小值。
- 当内容高度小于上限时，表格自然收缩，不留下明显空白。
- 当内容高度大于上限时，表格不再继续扩张，由 `QAbstractScrollArea` 自带滚动区域承载滚动。
- 当 `set_rows()` 被再次调用后，需要重新计算并更新高度，确保刷新后可增可减。

为避免引入额外容器，本次不新增外层“表格面板”组件，而是把受限高度能力直接放进现有通用表格类中，供多个弹窗复用。

### 2. 历史记录与已完成任务接入该能力

`[core/history_viewer.py](C:/Users/liang/Documents/solutions/task_manage/core/history_viewer.py)` 和 `[core/complete_table.py](C:/Users/liang/Documents/solutions/task_manage/core/complete_table.py)` 继续使用现有 `AdaptiveTextTableWidget`，只增加高度上限的接入逻辑。

接入方式如下：

- 由对话框在创建表格后，根据父级四象限窗口高度计算一个“建议最大表格高度”。
- 建议最大高度优先基于父级 `QuadrantWidget` 当前高度，扣除当前弹窗中的标题区、按钮区、面板内边距和必要留白。
- 若没有可用父级窗口，则回退到一个默认最大高度，避免独立打开或测试环境下报错。
- 历史记录弹窗保留现有字段合并、时间排序、导出逻辑不变，只调整表格尺寸控制。
- 已完成任务弹窗保留现有复选框列、还原按钮和刷新逻辑不变，只调整表格尺寸控制。
- 已完成任务刷新数据后仍应复用同一个表格实例，以满足现有测试对“面板内只有一张表格”的约束。

### 3. 颜色设置页重排为 `2 x 2` 网格

`[core/settings_dialog.py](C:/Users/liang/Documents/solutions/task_manage/core/settings_dialog.py)` 的颜色页从单列 `QFormLayout` 改为双列 `QGridLayout`。

布局规则如下：

- 网格固定为 `2 x 2`，四个格子分别对应 `q1`、`q2`、`q3`、`q4`。
- 每个格子是一个独立卡片，卡片标题使用当前象限名称，如“重要且紧急（右上）”。
- 每张卡片内部保留该象限完整颜色调节组，顺序保持一致：
  1. 颜色按钮
  2. 透明度滑杆
  3. 色相范围滑杆
  4. 饱和度范围滑杆
  5. 明度范围滑杆
- 卡片内部可继续使用表单式排版，但卡片外层要统一宽度、边距和间距，让四个象限的视觉结构对齐。
- 颜色按钮点击后的取色、滑杆值变化后的实时预览、`get_result()` 输出格式都保持不变。

### 4. 样式与视觉约束

为了避免颜色页只是“硬切成两列”而显得拥挤，需要增加少量结构样式：

- 颜色页卡片使用浅色背景、圆角边框和统一内边距，与当前设置面板风格保持一致。
- 卡片标题、说明文本和滑杆之间保留明确层次，避免视觉上混成一整块。
- 保持现有 settings panel 的总样式入口不变，尽量将新增样式限定在颜色页卡片内部，避免影响大小设置、界面设置和远程设置标签页。

## 高度计算与边界处理

### 高度来源

建议最大表格高度采用以下优先级：

1. 父级四象限窗口当前高度扣除弹窗其它区域后的可用高度。
2. 若无法获取父级或父级尺寸异常，则使用默认上限。

### 边界场景

1. 空数据或只有提示行时，表格应保持紧凑高度，不因默认上限而占据大块空白。
2. 数据量从多变少时，刷新后表格应重新收缩，不能卡在旧高度。
3. 父级窗口尺寸变化后，下一次打开弹窗时应使用新的可用高度重新计算。
4. 独立测试环境或无父级窗口场景下，不应因为缺少 `QuadrantWidget` 而出错。

## 测试策略

### 单元与组件测试

在现有测试基础上补充以下覆盖：

1. `AdaptiveTextTableWidget` 在设置高度上限后，会取内容高度与上限高度中的较小值。
2. `AdaptiveTextTableWidget` 在 `set_rows()` 后会重新计算高度，数据减少时会同步收缩。
3. 历史记录视图继续通过 `AdaptiveTextTableWidget` 创建表格，并设置受限高度能力。
4. 已完成任务弹窗刷新时继续复用同一张表格，并设置受限高度能力。
5. 设置对话框颜色页使用 `QGridLayout` 构成 `2 x 2` 结构，而不是单列顺序堆叠。
6. 每个象限卡片内仍包含颜色按钮、透明度、色相、饱和度、明度五类控件，且保留现有对象命名和结果写回逻辑。

### 手工验证

实现后需要人工确认：

1. 历史记录少量数据时弹窗不显得空。
2. 历史记录大量数据时表格内部出现滚动条，弹窗不继续拉高。
3. 已完成任务大量数据时行为与历史记录一致，且全选、取消全选、还原任务仍可正常使用。
4. 设置页颜色 tab 以 `2 x 2` 形式展示四个象限卡片。
5. 调整颜色按钮和滑杆时，四象限主窗口预览仍然实时更新。
