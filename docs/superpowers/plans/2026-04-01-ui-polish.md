# UI 视觉统一与交互打磨 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在不改动主要业务流程的前提下，为桌面端主界面、弹窗、表单、表格页和 gantt 页建立统一的视觉规范与轻量交互反馈，并明确排除“统一状态表达”的系统化重构。

**Architecture:** 以 [ui/styles.py](D:/repositories/task_manage/ui/styles.py) 为设计 token 和通用样式中心，以 [ui/ui.py](D:/repositories/task_manage/ui/ui.py) 为阴影与动效复用入口，逐步替换核心业务文件中的内联样式。桌面 PyQt 页面与 gantt web 页面共享同一套视觉方向，但保留各自技术栈内的最小实现差异。

**Tech Stack:** Python, PyQt6, Qt StyleSheet, embedded HTML/CSS/JavaScript, pytest/manual UI verification
